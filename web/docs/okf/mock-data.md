# Mock data

`src/mockData.ts` exports two seeds consumed by [`App.tsx`](/app-shell.md):

- `valleys: ValleyManifest[]` (`:3`) — currently one valley, **Escursione mattutina** (Italian Alps, bounds `6.746,44.892 → 6.822,44.934`). `assetBase: /assets/escursione-mattutina/`. Terrain is a 3000² grid over ~5988×4805 m, 1412–2637 m elevation, DEM = "Piemonte ICE 2009-2011 DTM 5m + IGN LiDAR HD fill". One route, 21.6 km / +1539 m / 30145 points.
- `initialShots: SavedShot[]` (`:67`) — three saved views (Generated final, Wide valley, Ridge context) with hand-tuned camera positions and texture modes.

# Schema

The inline `valleys` entry doubles as the fallback manifest, but [`TerrainViewer`](/terrain-viewer.md) re-fetches a `manifest.json` from `assetBase` at runtime and merges it (`TerrainViewer.tsx:299`). Expected asset files under `assetBase`:

```
manifest.json        ValleyManifest (authoritative terrain/route paths)
terrain.json         TerrainAsset heightfield
route.json           RouteAsset geometry
border.json          BorderAsset (optional, via overlays.border)
heightmap.png        + per-mode textures: terrain-texture, -topo-raw,
                     -hillshade, -multishade, -slope, -hypso, -normal
reference-*.png/jpg  reference render + preview
```

`attribution` lists the DEM source, source GPX (`Escursione_mattutina.gpx`), and that the reference render was produced by `plot3d_pv.py` — i.e. these assets are generated by an upstream Python pipeline outside this repo.
